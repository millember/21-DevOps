## 🎯 Что я научился делать

### DO1 — Linux
- Устанавливать Ubuntu Server без графического интерфейса, настраивать сеть, менять hostname, управлять временем.
- Создавать пользователей, добавлять их в группы, настраивать sudo.
- Работать с текстовыми редакторами VIM, Nano, Mcedit, редактировать конфиги, использовать поиск и замену.
- Настраивать SSH-сервер, менять порт, проверять подключения.
- Мониторить систему через top и htop: анализировать загрузку CPU, память, находить проблемные процессы.
- Анализировать диски через fdisk, df, du, ncdu.
- Читать системные логи, находить события авторизации.
- Настраивать cron для автоматического выполнения задач.

### DO2 — Linux Network
- Строить сеть из 5 виртуальных машин (3 рабочие станции + 2 роутера).
- Настраивать статическую маршрутизацию между разными подсетями.
- Поднимать DHCP-сервер для автоматической раздачи IP.
- Настраивать NAT и проброс портов через iptables.
- Анализировать трафик с помощью tcpdump, traceroute.
- Работать с SSH-туннелями (local, remote).

### DO5 — Simple Docker
- Писать Dockerfile для приложения на C + FastCGI и Nginx.
- Собирать образы, запускать контейнеры, пробрасывать порты.
- Использовать docker-compose для подъёма нескольких сервисов.
- Проверять безопасность образов через Dockle, исправлять предупреждения.
- Понимать разницу между `COPY` и `ADD`, `CMD` и `ENTRYPOINT`, слои образа.

### DO6 — CI/CD
- Устанавливать и регистрировать GitLab Runner.
- Писать `.gitlab-ci.yml` с этапами: сборка, стиль (clang-format), тесты, деплой.
- Настраивать артефакты, передачу файлов между джобами.
- Писать bash-скрипты для деплоя через SSH и SCP.
- Настраивать ручной запуск деплоя (`when: manual`) и уведомления (почта/Telegram).

## 🧠 Личные впечатления

Проекты Школы 21 дают не просто знания, а настоящее понимание того, как устроены операционные системы, сети и инфраструктура разработки.  
Каждый проект — это вызов, который требует самостоятельного поиска решений, отладки и умения читать документацию.

Особенно интересными были:
- **DO2** — когда впервые увидел, как пакет проходит через два роутера и доходит до нужной машины.
- **DO5** — когда написал Dockerfile и увидел, как приложение на C запускается в контейнере.
- **DO6** — когда нажал кнопку в GitLab, и через 10 секунд бинарники появились на удалённой машине.

Эти проекты заложили прочную базу для дальнейшего изучения DevOps, Kubernetes и облачных технологий.
