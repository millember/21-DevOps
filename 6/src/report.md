### Part 5. Этап деплоя

##### Клонировал вторую виртуальную машину *Ubuntu Server 22.04 LTS*.

![](./part5/1.png)

- удалил gitlab-runner на второй VM с помощью команды `sudo apt remove gitlab-runner`

![](./part5/2.png)

- настроил адаптер для обеих машин

![](./part5/3.png)
![](./part5/4.png)

- для машины, на которой стоит gitlab-runner я настроил через root пользователя `sudo su root` пароль для пользователя gitlab-runner с помощью команды `passwd gitlub-runner`. Это нужно, чтобы потом настраивать подключение к машине без gitlub-runner, а то будет требовать ввести несуществующий пароль.

![](./part5/5.png)

После этого надо на машине где gitlab-runner стоит перейти на юзера gitlab-runner:

`sudo su gitlab-runner`

Затем надо прожать:

`ssh-keygen`

![](./part5/6.png)

Затем ввести команду:

`ssh-copy-id [username]@10.10.0.2`

![](./part5/7.png)

На второй виртуальной машине, где удален gitlab-runner мы даем права (не через root, чтобы получить права в этой папке):

`sudo chmod 777 /usr/local/bin`

![](./part5/8.png)
